# Contributing to ObsShogi

ObsShogi に興味を持っていただきありがとうございます。

ObsShogi は、**定跡・序盤研究のワークフローに特化した将棋 GUI アプリ**です。  
局面検索、棋譜の切り替え、分岐の移動、USI エンジンによる検討を、研究の流れを止めずに行えることを目指しています。

このプロジェクトでは、Issue / Pull Request の両方を歓迎します。

---

## まず知っておいてほしいこと

ObsShogi はまだ発展途中のプロジェクトです。  
特に、開発者は **将棋AI・研究実務の現場でのユースケースをまだ十分に把握しきれていません**。

そのため、以下のような報告はとても助かります。

- 実際の研究で困ったこと
- 欲しい機能
- UI / UX 上のつまずき
- 棋譜管理や局面検索のワークフロー上の不満
- 不具合や再現手順
- USI エンジン連携で困るケース
- 将棋AI用途として不足している導線

「この機能がないと研究で使いにくい」
「こういうユースケースがある」
「この局面検索の仕様だと困る」

といった Issue は特に歓迎します。

---

## 参加方法

### 1. Issue を立てる

以下のような内容は、まず Issue で共有していただけるとうれしいです。

- バグ報告
- 機能提案
- 仕様の相談
- 研究用途・AI用途での要望
- ドキュメント改善案

Issue では、できるだけ具体的に書いていただけると助かります。

例:

- 何をしようとしたか
- 何が起きたか
- 本来どうなってほしかったか
- どの OS か
- 使っているエンジン
- 再現手順
- スクリーンショットやログ

### 2. Pull Request を送る

コードによる改善も歓迎します。

- バグ修正
- UI 改善
- ドキュメント改善
- テスト追加
- 開発体験の改善
- 将棋研究ワークフローに沿った機能追加

大きめの変更は、先に Issue で相談してもらえると助かります。  
方向性のずれを減らし、レビューもしやすくなります。

---

## 歓迎するコントリビューション

特に歓迎するもの:

- バグ修正
- 再現性のある不具合報告
- 研究ワークフローに基づく機能提案
- 将棋AI・USI エンジン利用者の観点からの改善提案
- キーボード操作・UI/UX 改善
- セットアップ手順や README の改善
- テストの追加
- エラーメッセージ改善

---

## Issue を立てる前に

重複を避けるため、既存の Issue を軽く確認してください。  
ただし、完全に同じか判断しづらい場合は、遠慮なく立てて大丈夫です。

---

## 開発の進め方

### セットアップ

```bash
npm install
npm run tauri dev
```

### 前提

- Rust (stable)
- Node.js (LTS 推奨)
- npm

---

## Pull Request のガイドライン

PR には、できるだけ以下を書いてください。

- 何を変更したか
- なぜ変更したか
- どの Issue に対応するか
- 影響範囲

### 小さく分けていただけると助かります

大きすぎる PR はレビューが難しくなるため、可能なら小さく分けてください。

### 設計方針について

ObsShogi は「研究のやりやすさ」を大事にしています。

- 研究の流れを止めないか
- キーボード中心の操作に合っているか
- 棋譜管理・局面検索・分岐移動と自然につながるか
- 将来的な拡張に無理がないか

を重視してレビューします。

---

## レビューについて

Pull Request は歓迎します。
メンテナが内容を確認し、**できるだけ積極的にレビュー**します。

ただし、以下の理由でそのまま取り込めないことがあります。

- プロジェクトの方向性とずれる
- UI / UX の一貫性を損なう
- 保守コストが高い
- 仕様の整理が不足している
- 先に Issue で議論したい

却下されることがあっても、提案自体はとてもありがたいです。

---

## Discussion / Issue の使い分け

- **Issue**: バグ、機能要望、作業として追いたいもの
- **Discussions**: アイデア相談、使い方、研究ワークフローの相談、雑談寄りの話題

迷ったら Issue でも大丈夫です。

---

## コミュニケーション

建設的で丁寧なやり取りをお願いします。
強い断定や攻撃的な言い方よりも、

- どう困っているか
- どう使いたいか
- どのような文脈で必要か

を書いてもらえると、より良い改善につながります。

---

## 最後に

ObsShogi は、既存の将棋 GUI と競争するというより、
**研究・序盤研究のワークフローをより良くする**ことを目指して育てているプロジェクトです。

将棋を研究する方、将棋AIを使う方、GUI を改善したいエンジニアの方、
Issue / Pull Request / Discussion いずれも歓迎します。
